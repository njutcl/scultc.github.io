实用小贴士

内建的图形化 git：
gitk
彩色的 git 输出：
git config color.ui true
显示历史记录时，每个提交的信息只显示一行：
git config format.pretty oneline
交互式添加文件到暂存区：
git add -i

在微信小程序开发的过程中接触到了这些概念（之前都是用GitHub网页版操作，很简单），所以必须学一下，感觉最常用的就是pull merge push fetch。第一次fetch仓库后要把head指向该仓库最后一次提交，然后pull和push。之后每次push之前，一定要先fetch，看看远程仓库有没有变化，因为如果有不一样的地方是不能直接push的。有两种选择：一是强推，二是pull，出现冲突就手动改一下，然后push。我还有一次是push之后把别人写的合并了，之前的又白写了，很绝望啊。
